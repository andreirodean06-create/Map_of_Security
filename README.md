# UVT Cybersecurity Knowledge Graph Vault

Welcome to the **UVT Cybersecurity Knowledge Graph Vault**. This project is a highly structured, interconnected digital notebook built explicitly for **Obsidian**. It maps out core cybersecurity domains, global threat landscapes, and academic tracking models, aligning them directly with the official curriculum, faculty expertise, and lab environments at the **West University of Timișoara (UVT)**.

By leveraging Obsidian's native linking capabilities ($[[Node_Name]]$), this vault visualizes security concepts not as isolated text blocks, but as a living, interconnected mesh of dependencies, research frontiers, and practical laboratory pipelines.

---

## Quick Start: How to Download and Open

To preserve the complete folder structure, custom formatting, and internal wiki-links, you must download this repository as a complete archive and open it natively inside Obsidian.

### Step 1: Download the Project
1. Click the green **`Code`** button at the top right of this repository page.
2. Select **`Download ZIP`** from the dropdown menu.
3. Once the download completes, extract the `.zip` archive to a permanent location on your local machine (e.g., your `Documents` or `Projects` directory).

### Step 2: Open in Obsidian
1. Download and install **[Obsidian](https://obsidian.md/)** (available for Windows, macOS, and Linux) if you haven't already.
2. Launch Obsidian. On the welcome screen, select **`Open folder as vault`** (click the **`Open`** button).
3. Browse to the directory where you extracted the ZIP file, select the root folder of this project, and click **Open**.

## Vault Architecture & Maps

The vault is divided into foundational security pillars, each containing an authoritative index map, subarea definitions, and verified local connections.

📂 UVT_Security_Vault/
├── 📂 01_Software_Security/         # Web, API, Client, and Secure Engineering
├── 📂 02_Cryptography/              # Primitives, PKI, Protocols, and Post-Quantum
├── 📂 03_Infrastructure_Security/   # Operating Systems, Networks, and Cloud
├── 📂 04_Threat_Intelligence/       # SOC, Incident Response, and Malware Analysis
├── 📂 05_Risk_Human_Security/       # Risk Governance, Compliance, and Human Factors
├── 📂 06_Problems/                  # Open Research Frontiers & Active Vulnerabilities
└── 📂 07_Directory/                 # Verified Faculty Profiles & Regional Venues

### Core Maps Included:
* **`Application & Software Security Map`**: Tracks application-layer interfaces, secure development lifecycles, and client fortification.
* **`Cryptography Map`**: Explores trust primitives, secure transport handshakes, and mathematical data protection boundaries.
* **`Systems & Infrastructure Security Map`**: Covers host operating system isolation, perimeter network routing, and container security.
* **`Security Operations & Threat Intelligence Map`**: Orchestrates enterprise event telemetry, binary reverse engineering, and threat campaign attribution.
* **`Governance, Risk & Human Security Map`**: Connects low-level technical vulnerabilities to macro-level business impacts, compliance metrics, and human factors.

## Recommended Navigation Tips

1.  **The Graph View**: Once the vault is open, press `Ctrl + G` (or `Cmd + G` on macOS) to open Obsidian's interactive Graph View. You can visually watch how abstract security domains gracefully link down to specific local UVT laboratory blue-prints and faculty nodes.
2.  **Unified Styling**: Every note uses structured markdown tables for core activities, unified callout blocks (`> [!warning]` or `> [!example]`) for open problems/labs, and direct cross-links to avoid orphaned pages.
3.  **Active Sandbox Modules**: At the bottom of each major security domain file, you will find an **Active UVT Project Sandbox**. These are step-by-step practical laboratory blueprints designed to test and validate theoretical concepts through actual implementation.
